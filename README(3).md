# Прогнозирование дефолта по истории кредитных продуктов

Бинарная классификация: по истории кредитных продуктов клиента предсказывается вероятность выхода в дефолт (просрочка более 90 дней в течение года). Целевая метрика — **ROC-AUC**.

## Структура данных

Каждому клиенту (`id`) соответствует несколько строк — кредитных продуктов из его истории, упорядоченных по полю `rn`. Все признаки представлены целочисленными кодами: категориальные признаки закодированы порядково и имеют небольшую кардинальность. Таргет задан на уровне клиента, а не отдельного продукта.

Такая структура «одна сущность — много упорядоченных записей» допускает два принципиально разных способа моделирования. В подходе используются оба, после чего их предсказания объединяются ансамблем — это и есть основная идея решения.

## Подготовка данных

### Представление 1 — агрегация в табличный вектор (для бустингов)

История продуктов каждого клиента сворачивается в один вектор признаков фиксированной длины. Для каждого исходного признака считаются статистики по всем продуктам клиента (среднее, стандартное отклонение, минимум, максимум, сумма, число уникальных значений), значение в последнем и первом продукте по порядку `rn`, а также общее число продуктов. Для категориальных признаков и флагов с ограниченной кардинальностью дополнительно добавляются частоты категорий — сколько раз каждое значение встретилось в истории клиента.

Поверх этих агрегатов строятся **тренд-признаки**, описывающие динамику внутри истории: разность «последнее значение минус первое» и «последнее значение минус среднее». Поскольку коды упорядочены (больший код примерно соответствует большему значению), такие разности осмысленны и улавливают ухудшение или улучшение профиля клиента со временем — что для скоринга является сильным сигналом.

### Отбор признаков

Из полученного набора удаляются константные признаки и один из каждой пары сильно коррелированных (выше заданного порога). Оставшиеся признаки ранжируются по важности (по бустингу) с опциональным отбором топ-K. Это снижает избыточность и шум, не теряя полезной информации.

### Представление 2 — последовательности (для нейросети)

Для нейросети история подаётся в исходном виде — как последовательность продуктов, упорядоченная по `rn`. Каждый продукт описывается набором категориальных кодов. Коды сдвигаются на +1, чтобы зарезервировать значение 0 под паддинг; на тесте значения, не встречавшиеся в обучающем словаре, обрезаются до известного диапазона; слишком длинные истории усекаются до последних N продуктов (самых свежих). Каждый категориальный признак получает собственный обучаемый эмбеддинг, и векторы признаков одного продукта конкатенируются. Так продукт превращается в плотный вектор, а клиент — в последовательность таких векторов, которую дальше обрабатывает рекуррентная или трансформерная модель.

## Модели

### Градиентный бустинг и линейный бейзлайн

На табличных (агрегированных) признаках обучаются три реализации градиентного бустинга — **LightGBM**, **XGBoost** и **CatBoost** — с ранней остановкой по валидационному AUC. Бустинги хорошо подходят для табличных данных: они устойчивы к нерелевантным признакам и автоматически улавливают нелинейности и взаимодействия между колонками.

В качестве линейного бейзлайна используется **логистическая регрессия** (с заполнением пропусков и стандартизацией признаков). Сама по себе она слабее бустингов, но как модель принципиально иной природы добавляет ансамблю разнообразия.

### Нейросеть на последовательностях

Модель `CreditSeqNet` работает напрямую с последовательностью продуктов и устроена так: каждый категориальный признак продукта проходит через свой эмбеддинг; эмбеддинги конкатенируются в вектор продукта; последовательность векторов кодируется энкодером; итоговое представление клиента подаётся в полносвязную «голову», которая выдаёт логит вероятности дефолта.

В роли энкодера выступает либо **двунаправленный GRU**, который накапливает историческое состояние от продукта к продукту, либо **Transformer**-энкодер. Обе версии используются как две самостоятельные базовые модели: они улавливают разные зависимости в истории, поэтому их совместное присутствие в ансамбле полезнее, чем любая из них по отдельности.

Ключевое преимущество этого представления в том, что нейросеть видит порядок и контекст продуктов, тогда как агрегация их «схлопывает» — поэтому сеть и бустинги дополняют друг друга.

## Обучение

### Кросс-валидация и out-of-fold предсказания

Всё обучение построено на едином разбиении `StratifiedKFold` со стратификацией по таргету. Одни и те же фолды используются для всех без исключения моделей — это обязательное условие корректного стекинга: предсказание каждой модели для каждого клиента получается на том фолде, где клиент находился в валидации (out-of-fold, OOF), без утечки таргета. Кроме OOF, каждая модель предсказывает весь тест, и эти тестовые предсказания усредняются по фолдам.

### Функция потерь нейросети

Помимо обычной бинарной кросс-энтропии (с повышенным весом редкого положительного класса для компенсации дисбаланса) нейросеть обучается с **попарным ранжирующим лоссом** (pairwise logistic loss). Он напрямую оптимизирует относительный порядок объектов — чтобы дефолтные клиенты получали более высокий скор, чем недефолтные, — что соответствует самой метрике ROC-AUC. Итоговая функция потерь — сумма кросс-энтропии и ранжирующего слагаемого; такая комбинация работает лучше каждого слагаемого по отдельности.

### Бэггинг по фолдам и усреднение по сидам

Нейросеть обучается на всех фолдах: для каждого фолда отдельная модель учится на обучающей части, формирует свою долю OOF и предсказывает весь тест; тестовые предсказания усредняются по фолдам (бэггинг снижает дисперсию и повышает стабильность). Дополнительно весь этот процесс повторяется с несколькими случайными сидами, а результаты усредняются. Усреднение по сидам особенно полезно для нейросети, поскольку она — самая сильная и одновременно самая «шумная» из базовых моделей, и сглаживание её предсказаний напрямую улучшает итоговый ансамбль.

## Объединение моделей: стекинг

Предсказания всех базовых моделей — трёх бустингов, линейного бейзлайна и обеих нейросетей — собираются в матрицу OOF-предсказаний, на которой обучается **мета-модель: логистическая регрессия** (стекинг). Мета-модель сама подбирает вес каждой базовой модели, опираясь на то, насколько её предсказания полезны. Так как OOF-предсказания получены без утечки, мета-модель учится на них корректно; для честной оценки качества стека её собственное качество дополнительно измеряется кросс-валидацией (`cross_val_predict`). Финальное предсказание на тесте — это обученная мета-модель, применённая к матрице тестовых предсказаний базовых моделей.

Предусмотрена подстраховка: если простое ранговое усреднение базовых моделей оказывается на OOF лучше стека (что иногда случается при малом числе коррелированных моделей), в качестве итогового берётся оно.

Идея ансамбля проста: модели разной природы — деревья, линейная модель, рекуррентная и трансформерная сети — ошибаются по-разному, и мета-модель извлекает пользу именно из их разнообразия, поэтому ансамбль почти всегда заметно сильнее любой одиночной модели. Наибольший вклад в итог вносит нейросеть на последовательностях, а бустинги добавляют дополнительный, частично независимый сигнал.

## Подбор гиперпараметров

Лучший по OOF AUC бустинг дополнительно оптимизируется с помощью **Optuna**. Дотюненная модель добавляется в ансамбль как ещё одна базовая, увеличивая его силу и разнообразие.
